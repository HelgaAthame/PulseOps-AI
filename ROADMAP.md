# PulseOps — Roadmap (Phase 2+)

> **Что это.** План расширения после завершённого 7-дневного MVP (`plan.md` — запись
> о MVP, все дни ✅ и задеплоено). Здесь — куда развивать продукт дальше.
>
> **Принципы приоритизации:**
> 1. **Усиливать продуктовую легенду** — micro-SaaS, AI-альтернатива ChartMogul для
>    indie-фаундеров на Stripe (см. позиционирование в README/памяти).
> 2. **Демонстрировать инженерную глубину** — то, что рекрутер/лид оценит как
>    senior-сигнал, а не «ещё одна CRUD-страница».
> 3. **Только бесплатные сервисы** (Stripe test mode, Upstash free, Resend free и т.п.).
>
> Легенда: 🔥 высокий приоритет · ⭐ средний · 💤 когда-нибудь · ✅ сделано.

---

## 🎯 Рекомендуемые следующие 3 (максимум отдачи)

1. **Реальный приём вебхуков Stripe** (🔥, Phase 1) — превращает легенду «connect
   Stripe» из копирайта в правду. Огромная прибавка к достоверности продукта.
2. **MRR movement waterfall** (🔥, Phase 2) — new / expansion / contraction /
   churned MRR. ЭТО фича, которой известен ChartMogul; даёт «parity» с лидером.
3. **Cohort retention** (🔥, Phase 2) — когортная кривая удержания. Фирменная
   SaaS-аналитика, сразу читается как «серьёзный продукт».

Эти три закрывают и продукт (parity + честная интеграция), и портфолио (нетривиальная
аналитика + вебхуки с верификацией подписи).

## 📌 Заметки

- **Порядок не жёсткий** — можно брать любой пункт под настроение/вакансию (например,
  под фронтенд-роль — Phase 2 + 6; под фуллстек — Phase 1 + 4).
- **Каждая крупная фича = отдельная ветка + PR** (даже соло) — это тоже портфолио-сигнал
  (чистая история, описания PR).
- **Не ломать легенду:** любые «данные» в демо остаются синтетическими; реальная
  Stripe-интеграция — через test mode.
- Обновлять этот файл как живой roadmap: помечать ✅ по мере реализации, переносить
  выполненное в `plan.md`/README при необходимости.
